【DeepSeek 第 10 问答复】前端 API 路径修复 + UI 优化 + 部署验证

Cursor 已修复 web_frontend 并部署至 ECS，摘要如下。

────────────────────────────────────────

一、问题背景

- 详情页曾请求错误路径 /api/v1/tasks/result?tid= → 404
- 后端正确路径：GET /api/v1/tasks/:tid
- 火焰图 iframe 曾指向 http://localhost:9000/... → 浏览器无法加载
- 详情页长期显示「加载中…」（API 失败或未处理 error）

────────────────────────────────────────

二、API 路径修复（web_frontend/src/api.js）

| 错误 | 正确 |
|------|------|
| /api/v1/tasks/result?tid= | GET /api/v1/tasks/:tid |
| /api/v1/tasks/flamegraph?tid= | /artifacts/:tid/flamegraph.svg（nginx 反代 MinIO） |

关键代码：
```javascript
export const getTask = (tid) => api.get(`/tasks/${tid}`).then((r) => r.data)
export const runAnalyze = (tid) => api.post(`/tasks/${tid}/analyze`).then((r) => r.data)
export const artifactUrl = (tid, name) => `/artifacts/${tid}/${name}`
```

路由支持：
- /task/:tid
- /task/result?tid=xxx（兼容旧链接，从 query 读 tid）

────────────────────────────────────────

三、UI 优化

首页：
- Hero +「新建采样」弹窗（紧凑表单，采集器默认 perf）
- Agent 列表：绿/灰圆点区分在线/离线，格式化最后心跳时间

任务列表：
- 状态彩色标签：PENDING 黄 / RUNNING 蓝 / DONE 绿 / FAILED 红

任务详情：
- 顶部 TID、状态、创建时间
- 三 Tab：火焰图 / TopN 热点 / 归因建议
- loading 动画 + error 提示
- 火焰图 iframe → /artifacts/{tid}/flamegraph.svg
- TopN 拉取 /artifacts/{tid}/top.json

nginx.conf 新增：
```nginx
location /artifacts/ {
    proxy_pass http://minio:9000/drop/;
}
```
MinIO bucket drop 已设 anonymous download。
