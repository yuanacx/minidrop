三、群聊新信息（Daniel Zhao）与我们的差距

群聊要点（Mini-Drop 相关，非 repm-tool）：

1. **apiserver 与 drop_server 可合并** → 我们分开，**不扣分**，功能满足即可
2. **前后端同仓、目录灵活** → 我们符合
3. **Agent 必须是可独立部署的 standalone binary** → 我们是 **Python 脚本 + Docker**，不是编译好的 drop_agent 二进制 → **与群聊强调有差距**；若评审严格按「二进制」查，需补 PyInstaller 或改 C++ agent
4. **多 Agent 注册** → drop_server 内存支持多 agent，compose 只跑 1 个 → 演示够用，答辩可能被问
5. **前端实时展示** → 我们轮询，非 WebSocket → 一般可接受
6. **perf 是核心必做** → ✅ 已通
7. **文档 + GitHub/Gitee + 石墨文档链接** → 代码仓有，**石墨提交链接需学生自己填**
8. **本地 Windows/Mac 开发也可** → 我们 ECS 部署，合理
9. 提到 **Mini-Drop 评分标准文档** → 建议对照该文档逐项打勾（本地未见该文件，可能在群文件/石墨）
